# User Preferences

## Communication Style
- Not very technical — prefers plain English explanations
- Wants agent to decide most things and just inform

## Code Style Preferences
- Trusts agent recommendations for best practices

## Design Preferences
- No hardcoded colors; theme tokens only
- No emoji characters in web projects; use SVG icon libraries (Lucide)
- Always use royalty-free stock images for hero sections
- Match visual style to business topic (AI/creative content domain)
- Be creative with design; avoid cookie-cutter templates

## Workflow Preferences
- Decide most things for him, just explain what was chosen
- Launch-ready build mode — solid foundation for real users from day one

## Custom Rules
- Queue system must cap platform costs at a predictable maximum
- Subscription model: credits recharge monthly, stack, lost on cancel
- Higher tiers get priority queue placement
